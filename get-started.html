Installation安装指南

Pre-GA · closed beta · invite only GA 前 · 闭测 · 仅限邀请

Get Sieve running 让 Sieve 跑起来

Sieve is a fully local LLM-traffic security proxy — a single Rust binary that sits between your AI coding agent and the upstream model API. This page covers how it ships: a signed .dmg, cosign-verified, dragged into /Applications. No remote scripts. Ever. Sieve 是一个完全本地运行的 LLM 流量安全代理 —— 一个 Rust 单二进制，夹在你的 AI 编码 agent 与上游模型 API 之间。本页讲清它如何分发：签名的 .dmg、用 cosign 验签、拖入 /Applications。不跑任何远程脚本，永远不跑。

macOS · Apple Silicon + Intel · Phase 1: macOS onlyPhase 1：仅 macOS · Apache-2.0

Builds are invite-only while we tighten the detection ruleset against real attack-replay samples. The source is public — SieveAI-dev/sieve — so you can verify rather than merely trust, but signed release artifacts are not yet generally available. The path安装路径

Five verifiable steps 五个可验证步骤

Every step is something you can inspect with your own eyes. Nothing runs that you didn't read first. 每一步你都能亲眼检查。不会运行任何你没先读过的东西。

STEP 01

Download the signed build 下载签名构建

Grab Sieve-<version>.dmg from the GitHub Releases page. Every release artifact is signed and accompanied by its cosign signature and certificate. 从 GitHub Releases 页面下载 Sieve-<version>.dmg。每个发布产物都已签名，并附带对应的 cosign 签名与证书。

github.com/SieveAI-dev/sieve/releases
# download from the Releases page (browser or gh CLI)
gh release download --repo SieveAI-dev/sieve \
   --pattern 'Sieve-*.dmg' \
   --pattern 'Sieve-*.dmg.sig' \
   --pattern 'Sieve-*.dmg.pem'
STEP 02 · REQUIRED必做

Verify the signature with cosign 用 cosign 验证签名

Do not skip this. Verify the .dmg against its Sigstore signature before mounting it. If verification fails, delete the file — a signature mismatch means you do not have an authentic Sieve build. 不要跳过这一步。挂载 .dmg 之前，先用 Sigstore 签名验证它。验证失败就删掉文件 —— 签名不匹配意味着你拿到的不是正版 Sieve 构建。

verify before you open
# keyless verification against Sieve's release identity
cosign verify-blob \
  --certificate          Sieve-1.0.0.dmg.pem \
  --signature            Sieve-1.0.0.dmg.sig \
  --certificate-identity-regexp '.*SieveAI-dev/sieve.*' \
  --certificate-oidc-issuer      'https://token.actions.githubusercontent.com' \
  Sieve-1.0.0.dmg

# expect: Verified OK
STEP 03

Drag into /Applications 拖入 /Applications

Mount the verified .dmg and drag Sieve.app into /Applications, exactly like any other macOS app. The bundle ships both the GUI and the sieve command-line tool. 挂载已验证的 .dmg，把 Sieve.app 拖进 /Applications，和任何其他 macOS 应用一样。这个 bundle 同时包含 GUI 和 sieve 命令行工具。

  • Notarized by Apple — Gatekeeper opens it without a security warning. 已经 Apple 公证 —— Gatekeeper 直接打开，无安全警告。
  • CLI is available as sieve after first launch. 首次启动后 sieve 命令即可用。
STEP 04

Run sieve setup 运行 sieve setup

One command wires Sieve into your agent: it detects Claude Code / Codex CLI / Cursor, points their base URL at the local proxy on 127.0.0.1, and loads the daemon. Your config is backed up first, and the change is idempotent. 一条命令把 Sieve 接入你的 agent：它检测 Claude Code / Codex CLI / Cursor，把它们的 base URL 指向 127.0.0.1 上的本地代理，并加载 daemon。修改前先备份你的配置，且操作幂等。

~ — zsh
sieve setup

# [✓] Claude Code detected → base_url rewritten
# [✓] config backed up to ~/.sieve/backups/
# [✓] daemon loaded on 127.0.0.1:11453
STEP 05

Check health, then watch decisions 体检，然后盯住决策流

sieve doctor runs a full self-check — listeners bound, upstream reachability, TLS, hook registration. Once it's green, sieve decisions watch streams every decision Sieve makes in real time, so you can see redactions and Critical interceptions as they happen. sieve doctor 跑一次完整自检 —— listener 已绑定、上游可达、TLS、hook 已注册。全绿之后，sieve decisions watch 实时输出 Sieve 做的每一个决策，让你能看到脱敏与 Critical 拦截在发生时的样子。

sieve doctor
sieve doctor

# [✓] daemon running (pid 4821)
# [✓] listener 127.0.0.1:11453 LISTEN
# [✓] upstream api.anthropic.com reachable
# [✓] PreToolUse hook registered
# [✓] ruleset loaded · signature OK
sieve decisions watch
sieve decisions watch

14:02:40  allow    read_file

Deliberate omission刻意不做

Why there is no curl | sh 为什么没有 curl | sh 一键脚本

A one-line install script asks you to pipe a remote, unread file straight into your shell with your full privileges. That is exactly the attack surface Sieve exists to close: blindly executing something you never inspected, fetched from a server you don't control, that can change between the moment a blog post was written and the moment you run it. 一键安装脚本要求你把一个远程的、你从没读过的文件直接管道进 shell，带着你的全部权限运行。这恰恰是 Sieve 存在的意义所在 —— 要堵住的攻击面：盲目执行你从未检查过、从你无法控制的服务器拉取、且在博客写下与你运行之间可能被悄悄替换的东西。

A security product cannot, in good conscience, ask you to do the unsafe thing on day one. So Sieve ships a signed, notarized, cosign-verifiable .dmg instead — slower by a few steps, honest by design. 一个安全产品没法昧着良心，在第一天就要你去做那件不安全的事。所以 Sieve 改为分发签名、公证、可用 cosign 验签的 .dmg —— 多几步、慢一点，但设计上是诚实的。

curl … | sh
Run unread remote code. Not offered. 盲跑远程代码。不提供。
cosign verify-blob
Prove authenticity before anything runs. 运行任何东西之前先证明真实性。
signed .dmg
Apple-notarized, Sigstore-signed artifact. Apple 公证 + Sigstore 签名的产物。

Configuration配置

Multiple upstreams, one binary 多上游，单二进制

sieve setup writes a working config for you, but you can edit it directly. Sieve uses port-based listeners: each [[upstream]] block binds a local port to one upstream, so a "dumb" client that only knows a single base URL can still reach several providers by pointing at different ports. sieve setup 会替你写好一份能用的配置，你也可以直接编辑它。Sieve 采用基于端口的 listener：每个 [[upstream]] 块把一个本地端口绑到一个上游，于是只认单个 base URL 的「哑」客户端也能通过指向不同端口，访问多个 provider。

~/.sieve/config.toml
# Each listener binds 127.0.0.1:<port> to one upstream.
# bind address is always loopback — never configurable.

# default listener — Anthropic on the canonical port
[[upstream]]
port        = 11453
url         = "https://api.anthropic.com"
provider_id = "anthropic"
protocol    = "anthropic"

# a second upstream speaking the Anthropic protocol,
# with an upstream path prefix that is preserved
[[upstream]]
port        = 11454
url         = "https://api.deepseek.com/anthropic"
provider_id = "deepseek"
protocol    = "anthropic"

# an OpenAI-protocol upstream on its own port
[[upstream]]
port        = 11455
url         = "https://api.openai.com"
provider_id = "openai"
protocol    = "openai"

Field reference 字段说明

  • port Local listen port. Always bound to 127.0.0.1 — the bind address is a hard invariant and cannot be changed.本地监听端口。永远绑定到 127.0.0.1 —— 绑定地址是硬不变量，不可更改。
  • url The real upstream, including any path prefix (e.g. /anthropic), which is preserved on forward.真实上游，含任何 path 前缀（如 /anthropic），转发时保留。
  • provider_id A label used in audit records, IPC events and logs — purely for your own attribution.用于审计记录、IPC 事件与日志的标签 —— 纯粹方便你自己归因。
  • protocol Explicitly "anthropic" or "openai". A declared protocol enables fail-closed rejection on path mismatch — Sieve never guesses.显式声明 "anthropic" 或 "openai"。声明后，path 错位会触发 fail-closed 拒绝 —— Sieve 绝不靠猜。
Point your agent at the matching port, e.g. ANTHROPIC_BASE_URL=http://127.0.0.1:11453. sieve setup does this automatically. 把你的 agent 指向对应端口，例如 ANTHROPIC_BASE_URL=http://127.0.0.1:11453。sieve setup 会自动完成这件事。

Sieve

A fully local LLM-traffic security proxy for crypto-native developers. Inspects traffic in both directions — redacting secrets outbound, blocking dangerous tool calls inbound, fail-closed. 面向 crypto 开发者的完全本地 LLM 流量安全代理。双向检测流量 —— 出站脱敏密钥、入站拦截危险工具调用，fail-closed。
