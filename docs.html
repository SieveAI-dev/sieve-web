03 — FAQ03 — 常见问题

Frequently asked questions常见问题

Can Sieve see my prompts?Sieve 会看到我的 prompt 吗？

Detection runs entirely on your machine. Your prompts and the model's responses are inspected locally and are never uploaded anywhere for analysis. The traffic still goes to whatever upstream you point at (Anthropic / OpenAI / your relay) — exactly as it would without Sieve — but Sieve itself adds no cloud step. 检测完全在你的机器上运行。你的 prompt 与模型响应在本地被检查，永不上传到任何地方做分析。流量仍会发往你指定的上游（Anthropic / OpenAI / 你的中转站）——和没有 Sieve 时完全一样——但 Sieve 本身不引入任何云端环节。

Which agents and upstreams are supported?支持哪些 agent 与上游？

Agents that speak a standard model API work — Claude Code, Codex, and Cursor among them. Upstreams include Anthropic, OpenAI, and relay endpoints. You point each agent's base URL at a local Sieve listener and route it to the upstream of your choice. 说标准模型 API 的 agent 都能用——Claude Code、Codex、Cursor 都在其中。上游包括 Anthropic、OpenAI 以及中转站（relay）端点。你把每个 agent 的 base URL 指向本地 Sieve listener，再路由到你选定的上游。

How is this different from a wallet-security product or DLP?和钱包安全产品、DLP 有什么区别？

A wallet-security product guards the signing layer; DLP guards files and network egress broadly. Sieve sits at a different point: the LLM traffic between your coding agent and the model. It redacts secrets the model would otherwise see (outbound) and intercepts dangerous tool calls the model returns (inbound, fail-closed). It complements — not replaces — your wallet and DLP. 钱包安全产品守护签名层；DLP 宽泛地守护文件与网络出口。Sieve 在另一个位置：编码 agent 与模型之间的 LLM 流量。它脱敏模型本会看到的密钥（出站），并拦截模型返回的危险工具调用（入站、fail-closed）。它是对钱包和 DLP 的补充，而非替代。

Can I turn off Critical interception?Critical 能关吗？

No. Critical interception cannot be disabled in any mode. Irreversible actions — signing, transfers, deploys — are always held for human confirmation (HIPS), and if detection is uncertain it fails closed. This is the core safety guarantee and is non-negotiable. 不能。Critical 拦截在任何模式下都不可关闭。不可逆操作——签名、转账、部署——始终被扣住等待人工确认（HIPS），若检测不确定则 fail-closed。这是核心安全保证，没有商量余地。

How do I turn off telemetry?怎么关遥测？

Set an environment variable. Telemetry rides the bounded update channel, so disabling either turns it off: 设一个环境变量即可。遥测搭载在边界清晰的更新通道上，关掉任一即可禁用：

SIEVE_NO_TELEMETRY=1 disable telemetry禁用遥测
SIEVE_NO_UPDATE=1 disable update checks (and the telemetry that rides them) entirely完全禁用更新检查（连同搭载其上的遥测）

Is Sieve open source?Sieve 是开源吗？

The code is open source under Apache-2.0. The documentation is licensed CC BY-NC-SA 4.0. Both repositories live under the SieveAI-dev organization on GitHub. 代码以 Apache-2.0 开源。文档采用 CC BY-NC-SA 4.0 许可。两个仓库都在 GitHub 的 SieveAI-dev 组织下。

Do you support Linux and Windows?支持 Linux / Windows 吗？

Today the GUI ships for macOS. Linux and Windows are planned for Phase 2. The daemon is a portable Rust binary, so cross-platform support follows the GUI work. 目前 GUI 仅发布 macOS 版。Linux 与 Windows 计划在 Phase 2 支持。守护进程是可移植的 Rust 二进制，跨平台支持随 GUI 工作推进。

Why is there no curl | sh installer?为什么没有 curl | sh？

A security product should never ask you to blindly pipe a remote script into your shell. Doing so executes whatever the server returns, sight unseen — the exact habit Sieve exists to break. Installation uses inspectable, verifiable steps instead. See Get started开始使用. 安全产品绝不该让你把远程脚本盲管道进 shell。那会原样执行服务器返回的任何东西、毫无审视——正是 Sieve 要打破的习惯。安装改用可审视、可验证的步骤。见 Get started开始使用。

Open an issue on GitHub, or read the source directly. The deep docs are versioned alongside the code. 在 GitHub 提 issue，或直接读源码。深入文档与代码一同版本化。

Built for crypto-native developers. 一个完全本地的 LLM 流量安全代理。它夹在你的 AI 编码 agent 与模型 API 之间——出站脱敏密钥，入站阻断危险工具调用（fail-closed）。为 crypto-native 开发者而建。
